# Contributing to Kavosh
=====================================

Thank you for considering contributing to Kavosh! We appreciate your enthusiasm and expertise. This guide outlines the steps to contribute to our project.

## Forking the Repository
-------------------------

To start contributing, fork the Kavosh repository:

1. Go to the [Kavosh repository](https://github.com/kavosh/kavosh) on GitHub.
2. Click the **Fork** button in the top-right corner.
3. Choose your GitHub account or organization to fork the repository into.
4. Wait for the fork to complete.

## Branch Naming
----------------

When creating a new branch, use the following naming convention:

* `feature/<feature-name>` for new features
* `fix/<issue-number>` for bug fixes
* `docs/<documentation-change>` for documentation updates
* `refactor/<code-change>` for code refactoring

Replace `<feature-name>`, `<issue-number>`, `<documentation-change>`, and `<code-change>` with descriptive and concise names.

## Commit Style
----------------

Follow these guidelines for commit messages:

* Use the imperative mood (e.g., "Add new feature" instead of "Added new feature")
* Keep the first line concise (less than 50 characters) and descriptive
* Use a blank line to separate the summary from the body
* Use bullet points in the body for multiple changes or highlights

Example:
```markdown
Add new login feature

* Implement login functionality
* Update authentication API
* Fix related issues
```

## Pull Request Template
-------------------------

When creating a pull request, use the following template:

### Pull Request Title
* Briefly describe the changes made

### Pull Request Description
* Describe the changes made and the purpose of the pull request
* Mention any related issues or discussions
* List any changes or highlights

Example:
```markdown
### Pull Request Title
Add new login feature

### Pull Request Description
This pull request implements a new login feature, including authentication API updates and bug fixes.

* Related issue: #123
* Changes:
	+ Implemented login functionality
	+ Updated authentication API
	+ Fixed related issues
```

## Code of Conduct
-----------------

By contributing to Kavosh, you agree to abide by our [Code of Conduct](CODE_OF_CONDUCT.md). Please review it before contributing.
